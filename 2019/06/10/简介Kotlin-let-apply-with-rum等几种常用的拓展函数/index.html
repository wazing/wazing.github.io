简介Kotlin let,run,also,apply,with等几种常用的拓展函数

发表于 2019-06-10 | 分类于 Kotlin | 阅读次数:

Kotlin的Standard Library提供了几种拓展函数(extension function)，有的可以更优雅处理可空变量问题。

相信在学习 Kotlin 时，一定都会注意到它有许多let apply run 等这些 extension function，它们看起来大同小异，用起来似乎也经常可以相互替换，稍做修改就让程式依照正确的逻辑执行。

function identifier return
Let it 最后一行
run This 最后一行
also it this
apply this this

常用

T.let

T是什么?

在Kotlin extension function中需要接在一个变数后面才行。像是 someVariable.run { /* do something */ }，包含 T.run 下面四个 let also apply 都属于这种 extension function。因为 run有两种用法，这里为了避免混淆而将 T 写出来。

T.let 默认当前这个对象作为闭包的it参数，返回值是函数里面最后一行，或者指定return@let

结果: my name is wazing

  1. name.let{} 如果name为null，let{}中的程式码还是会被执行，只不过此时it为null;
  2. name?.let{} 如果 name为null，let{} 中的程式码不会执行；
  3. T 在 scope 内则是用 it 来存取而不是 this。也可以依照需求改成其他的名字，增加可读性；
  4. T.let{}会作为下一个扩展函数或者回传给之前的变数。

Run

同样如果传进来的变数是空值，T.run{} 内的程式码就根本不会执行了。除此之外， T.run 和 run 的特性完全一样。可以将最后一行的结果回传，或是作为下个extension function。

T在T.run{}与T.let的scope内相反，T.run{}是用this而不是 it。

also

可以写作T.also

与上面的run与let的不同之处在于: run与let会将最后一行传给下个extension function或是回传；also和apply则是将「自己(this)」回传或传入下个extension function。

结果: [苹果🍎, 梨子🍐, 菠萝🍍, 西瓜🍉]

apply

可以写作T.apply

apply与also相似，不同的地方是apply在 scope内T的存取方式是this，其他都与also`一样。
