Git指南

发表于 2019-08-18 | 分类于 Android | 阅读次数:

前言

Git 是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。Git是Linus Torvalds为了帮助管理Linux内核开发而开发的一个开放源码的版本控制软件。

GitHub 是最大的 Git 版本库托管商，是成千上万的开发者和项目能够合作进行的中心。 大部分 Git 版本库都托管在 GitHub，很多开源项目使用 GitHub 实现 Git 托管、问题追踪、代码审查以及其它事情。

配置

安装完成后，需要配置用户名和邮箱地址，因为这些内容会出现在你的每一个提交（commit）里面的;

配置用户名和密码

Git的配置信息分为全局和项目两种，上面命令中带了"--global"参数，这就意味是在进行全局配置，它会影响本机上的每个一个Git项目。

不需要全局配置的去掉上边的"--global"即可(公司项目和个人开源项目会有该需求);

GitHub 使用用户邮件地址区分 Git 提交。 如果你在自己的提交中使用了多个邮件地址，希望 GitHub 可以正确地将它们连接起来，你需要在管理页面的 Emails 部分添加你拥有的所有邮箱地址。

查看用户名和邮箱

或者可以通过.gitconfig文件查看，该文件一般在个人文件根目录下

创建github帐号

GitHub是一个面向开源及私有软件项目的托管平台，因为只支持git 作为唯一的版本库格式进行托管，故名GitHub。

  1. 打开官网：https://github.com 右上角 sign in or sign up 分别为登录或注册，点击注册 后，需要填写一个 username(用户名) 、email(邮箱地址) 和 password(密码) 填写完成后一路提交。
  2. 注册完成后，点击头像—setting—SSH and GPG keys—New SSH key

  3. 等待下边配置ssh key, 后面创建生成id_rsa.pub文件后，将该文件内容复制到key中

配置ssh key

SSH 为 Secure Shell 的缩写，由 IETF 的网络小组（Network Working Group）所制定。利用 SSH 协议可以有效防止远程管理过程中的信息泄露问题。简单说，SSH是一种网络协议，用于计算机之间的加密登录。

  1. 检查是否存在

如果存在，先将已有的ssh备份，或者将新建的ssh生成到另外的目录下，如果不存在，通过默认的参数直接生成ssh

  1. 生成ssh key

期间会提示你输入key的文件名称, 密码和确认密码, 一路Enter

完成后会在个人文件夹根目录有一个.ssh文件夹, 打开后有两个文件 id_rsa和id_rsa.pub

  1. 将ssh key保存在github中

打开id_rsa.pub文件, 将里边的内容复制到配置ssh key, title代表别名, 用作区分, 可不填可填任意字符

git的基础使用

git的基础使用很简单，比如在github中创建一个空的仓库就会有相应的命令提示:

  1. echo “# test” >> README.md

    终端的命令，表示创建一个README.md文件并写入字符# test；

  2. git init

    创建新的 Git 仓库。 你可以在任何时候、任何目录中这么做，完全是本地化的；

  3. git add README.md

    将文件添加到本地缓存中，README.md就是刚刚创建的文件；

  4. git reset HEAD

    撤销已在缓存区的文件，如：git reset HEAD -- README.md 撤销刚刚添加在本地缓存中的README.md文件;

  5. git status

    查看在你上次提交之后是否有修改。如果在后面追加-s则是查看简短信息，反之查看详细信息。

  6. git commit

    将缓存区内容添加到仓库中。git commit -m "Add README.md" #后面的文本为提交日志，建议每次提交到仓库都要写提交日志。

  7. git pull origin master

    把远程仓库代码合并，以保证两端同步，建议每次提交前先同步代码；

  8. git push origin master

    把本地代码提交到github仓库master分支。一般在push前都会先pull，这样不容易造成冲突。

  9. git clone

    拷贝一个Git仓库到本地，让自己能够查看该项目，或者进行修改。如果你需要与他人合作一个项目，或者想要复制一个项目，看看代码，你就可以克隆那个项目。 比如：
